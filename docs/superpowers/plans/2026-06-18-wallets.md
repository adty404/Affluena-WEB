## Scope Deviations from Original Mock

The mock (`src/data/mockWallets.ts` pre-Plan-2) carried fields backend does not provide:

| Field | Mock | Backend | Resolution |
|-------|------|---------|------------|
| `balance` | number | `balance_minor` | Renamed. UI uses `<Amount value={balance_minor} />` |
| `currency` | `'IDR'` | `currency_code` | Renamed |
| `color` | green/blue/... | (none) | Removed. Card derives color from `type` |
| `isShared` | boolean | derived from `role !== 'owner'` | Computed in `WalletCard` |
| `memberCount` | number | (none) | Removed. Show "Shared/Private" label only |
| `monthlyInflow`/`monthlyOutflow` | number | (none â€” needs dashboard endpoints) | Removed from card + detail page |
| `lastActivity` | string | (none â€” needs transactions) | Removed |
| `description` | string | (none) | Removed |
| `members: [{role,status}]` | array | (none) | Removed. Sharing page shows current user's role/status + invite form only |

Mock file (`src/data/mockWallets.ts`) is **kept** because `DebtFormPage`, `GoalContributionPage`, `GoalFormPage`, `InstallmentPaymentPage`, and `SubscriptionPaymentPage` still import it for their wallet-select dropdowns. Those pages will drop the import when their respective plans (Plan 7, Plan 8) land. The mock itself is rewritten to match the new `Wallet` shape so it type-checks.

## Known Limitations

- **No members list endpoint**: sharing page cannot show all members of a shared wallet. Backend `GET /wallets/:id/members` does not exist. Workaround: only show current user's role/status.
- **No monthly inflow/outflow at wallet level**: detail page drops the chart. To re-add, scope a `GET /wallets/:id/analytics?month=YYYY-MM` endpoint in a later backend sprint.
- **No transaction history on detail page**: shows placeholder text. Plan 4 (Transactions) will wire `GET /api/v1/transactions?wallet_id=...`.
- **Mock `mockWallets.ts` retained**: cross-plan consumers exist. Remove after Plans 7 + 8 land.
